Animation card : step-1

안녕하세요. 여기서는 모바일에서 touch하거나, 마우스 커서를 hover했을 때, 어떤 애니메이션이 동작하는 온라인 카드를 만들어 봅니다. 온라인 카드는 크리스마스나, 신년, 생일, 기념일을 포함해서 실생활에서 쓰임이 많습니다. 종이 카드를 한글자 한글자 정성들여서 쓰던 것처럼, 코드를 한줄 한줄 정성들여 온라인 수제 카드를 만들어 봅시다.
받는 사람이 어떤 시나리오로 카드를 받게 될까요?

  1. 받는 사람에게 URL 주소가 담긴 카톡 메시지를 전달하게 될 것입니다.
  2. 받는 사람은 메시지를 스마트폰으로 볼 수도 있고, 컴퓨터에서 볼 수도 있습니다.
  3. 받는 사람은 메시지를 확인하고, 클릭해서 웹 브라우져를 열게 됩니다.
  4. 그리고 우리가 만든 카드를 보게 됩니다.
  5. 카드를 터치하면 커버가 스르륵 밀려나가면서 내용이 보이도록 합니다.

이정도 시나리오를 상상해 볼 수 있겠네요. 따라서, 우리가 만들 웹 페이지는 URL로 보낼 수 있도록 해야하고, 스마트폰의 좁은 화면이나 컴퓨터 브라우저의 넓은 화면에서 모두 볼 수 있어야 할 것 같아요.

그리고, 사용자가 카드를 열어서 볼 수 있도록 인터렉션을 주어야하겠네요. 먼저 앞면과 뒷면을 겹치듯 배치하고, 터치하거나 마우스커서를 올리면 앞면이 밀리도록 해야겠습니다.

앞면/뒷면을 겹치고, 터치했을 때 스르륵하게 만듭니다.

  • HTML의 기본 코드를 세팅합니다. 가장 먼저 head태그 안에 title태그를 추가하여 이 문서의 제목을 작성합니다.

    우리가 위에 시나리오에서 살펴 봤듯이, 이번 웹 페이지는 모바일에서 볼 수 있게 됩니다. 따라서 제 기억으로 모바일에서 웹 페이지를 보기 위해서는 어떤 코드가 필요하다는 이야기를 들은것도 같은데, 저도 잘 기억이 안나니, 구글에 검색해 봅시다. 구글에는 html 모바일 최적화라고 검색합니다.

    head태그 안에 어떤 meta태그를 추가하라고 하는걸 찾으셨나요? 그대로 복붙 하겠습니다.

    See the Pen animation card - step 1 by los0731 (@los0731) on CodePen.

  • 우리는 두 장의 종이를 만들어야 합니다. 커버cover와 내지inner라고 합시다. 그 두 장의 종이가 하나의 카드인거죠. 따라서 하나의 div태그를 card라는 클레스로 묶고, 그 안에 또 다른 두 개의 div태그를 만들어서 각각 cover와 inner라는 클래스를 지정하겠습니다.

    물론 cover와 inner가 아직 카드 같이 보이진 않네요.

    See the Pen animation card - step 2 by los0731 (@los0731) on CodePen.

  • 두 장의 종이인 div태그에 넓이와, 높이와, 배경색을 지정합니다. 보통은 종이가 흰색이니까, 흰색이 돋보이도록 body의 배경색도 임시로 grey라는 컬러로 지정해 둡니다. 나중에 배경에는 멋진 이미지를 채우도록 할거니까요.

    card클래스의 바로 안에 있는 cover와 inner클래스에게 어떤 속성을 부여할 겁니다. 여기서 기억 나나요? 맞아요. 이게 자식선택자라고 하죠.

    .class-a > .class-b {...}
    // class-a 바로 1단계 안에 있는 class-b만 선택합니다.
    이렇게 작성합니다. 그렇다면 하위선택자도 기억나나요? 하위선택자는 아래와 같이 작성합니다.
    .class .class {...}
    // class-a 안에 있는 모든 class-b를 선택합니다.
    자식선택자와 하위선택자의 차이는 자식선택자는 바로 아래의 클래스만 선택되고, 하위선택자는 클레스 이름이 같다면 아래의 아래의 아래의 클레스도 선택합니다. 조금 더 자세히 알아보고 싶다면 선택자의 이해를 확인하세요. 참고로 어떤 곳에서는 하위선택자라고 하기도하고, 어떤 곳에서는 자손선택자라고 하기도 합니다. 뭐, 개념만 이해하면 됩니다.

    See the Pen animation card - step 3 by los0731 (@los0731) on CodePen.

  • 아직 카드 같진 않지만, 그래도 배경과 분리된 뭔가가 생겼네요. 계속 해봅시다.

    cover와 inner태그를 더 카드처럼 보이게 하기 위해서 넓이와 높이를 부여합니다. 그리고 카드는 웹 페이지의 중앙에 위치했으면 좋겠어요.

    그런데 위에 우리가 이야기 한대로 카드의 크기가 화면에 맞게 조절 되어야 합니다. 여러가지 방법이 있지만, 저는 넓이와 높이를 전체영역의 80%로 줄거에요. 대신 그렇게 되면, 큰 모니터에서는 카드가 엄청 커지겠죠? 그래서 아무리 커져도 480px보다는 커지지 않도록 할 겁니다. 코드를 봅시다.

    실행된 결과 창을 보면, 넓이는 지정한대로 된 것 같은데, 높이는 변함이 없네요. 일단 나둡니다.

    See the Pen animation card - step 4 by los0731 (@los0731) on CodePen.

  • 카드를 더 잘보기 위해서는 화면의 중앙에 위치시켜야 합니다. 두 종이의 position을 absolute로 지정하고, top, left를 이용해서 위치를 가운데로 옮겨 줍니다.

    position: absolute;의 속성으로 인하여 두 장의 카드는 서로 겹쳐지게 됩니다. html은 아래에 작성된 코드가 더 위에 표시되기 때문에, cover는 inner 아래에 깔려 있게 되는 거죠.

    top: 50%와 left: 50%;는 위에서부터 50%만큼, 왼쪽에서부터 50%만큼, 떨어지도록 합니다.

    여기까지 되었다면 카드의 위치가 가운데가 아닌 오른쪽/아래로 치우쳐 보일 텐데요. 딱 정확하게 위에서부터 50%만큼 떨어지도록 했기 때문이죠.

    See the Pen animation card - step 5 by los0731 (@los0731) on CodePen.

  • 카드가 화면 중앙에 있어 보이도록 하기 위해서는 left: 50%를 부여한 후에 카드 넓이의 절반 만큼 왼쪽으로 이동해야 합니다. 여기서 필요한 속성이 transform입니다.

    여기서, transform은 번역하면 변형시키다라는 뜻을 가지고 있는데, 여기서 변형이란 회전하고, 움직이게 하고, 크기를 확대/축소하는 물리적인 변형을 말합니다. 위에 작성한 top속성은 어떤 공간이나 요소를 기준으로 밀거나 당기는 속성이라고 한다면, transform은 자기 자신을 기준으로 변형합니다. 값은 아래와 같은 규칙으로 작성합니다. 

    transform: translateX(-50%), translateY(-50%;);
    변형해: X축으로옮겨(-50%만큼), Y축으로옯겨(-50%만큼);

    transform은 ,을 붙여서 한 번에 여러개의 속성값을 부여할수 있습니다. 이제 카드가 가운데로 위치했네요.

    See the Pen animation card - step 6 by los0731 (@los0731) on CodePen.

  • 을 보면 inner카드가 위에 덮어져 있습니다. 하지만 우리는 cover카드가 위에 덮어지도록 해야겠죠. 이렇게 위에 올라가 있거나 아래에 내려가 있도록 하는 속성을 z-index라고 합니다.

    z-index: 1;과 같이 양수로 적용하면, 적용되어 있지 않은 태그보다 상위에 위치합니다. 반대로 z-index: -1;처럼 음수를 적용하면 하위에 내려와서 위치합니다.

    그리고 z-index는 position이 설정되어 있어야만 동작합니다. 우리는 이미 position: absolute;로 지정했기 때문에 정상적으로 동작할 거예요. z-index에 대하여 더 자세히 알고 싶다면 MDN web docs를 확인하세요.

    cover에만 z-index를 부여합니다. cover가 위로 올라온걸 확인하셨다면 성공입니다.

    See the Pen animation card - step 7 by los0731 (@los0731) on CodePen.
